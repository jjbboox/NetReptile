# getimgfromurl.py 批量处理功能使用示例

## 新增功能

`getimgfromurl.py` 现在支持 `--list` 参数，允许从文件批量处理多个URL。

### 功能特点
1. **批量处理**：通过list文件一次处理多个URL
2. **灵活格式**：list文件每行包含URL和输出目录
3. **进度显示**：实时显示每个任务的进度和结果
4. **错误处理**：单个任务失败不会影响其他任务
5. **详细统计**：提供完整的处理统计信息

## 基本用法

### 1. 单URL模式（原有功能）
```bash
python getimgfromurl.py https://example.com ./output
python getimgfromurl.py https://example.com ./output --timeout 30000 --ext png --verbose
```

### 2. 批量处理模式（新增功能）
```bash
# 基本用法
python getimgfromurl.py --list urls.txt

# 使用自定义超时时间
python getimgfromurl.py --list urls.txt --timeout 60000

# 使用代理和自定义用户代理
python getimgfromurl.py --list urls.txt --proxy http://proxy.example.com:8080 --user-agent "Custom Agent"

# 启用详细日志
python getimgfromurl.py --list urls.txt --verbose
```

## List文件格式

List文件是一个文本文件，每行包含两个字段：
1. **URL**：要处理的网页URL
2. **输出目录**：保存图片的目录

### 格式示例
```
# 这是一个批量处理list文件示例
# 空行和以#开头的行会被忽略

# 示例1：简单URL和输出目录
https://example.com/page1 ./output/page1_images
https://example.com/page2 ./output/page2_images

# 示例2：相对URL（会自动添加https://）
example.com/page3 ./output/page3_images

# 示例3：使用制表符分隔
https://site1.com/gallery	./downloads/gallery1
https://site2.com/photos	./downloads/gallery2
```

### 注意事项
1. URL和输出目录之间可以用空格或制表符分隔
2. 如果URL不以http://或https://开头，程序会自动添加https://
3. 输出目录可以是相对路径或绝对路径
4. 程序会自动创建不存在的输出目录

## 输出示例

运行批量处理时，程序会显示：
1. 每个任务的开始和结束状态
2. 找到的图片数量
3. 成功下载的图片数量
4. 失败的任务详情
5. 总体统计信息

## 实际使用示例

### 步骤1：创建list文件 `my_urls.txt`
```
https://example.com/gallery1 ./downloads/gallery1
https://example.com/gallery2 ./downloads/gallery2
https://example.com/gallery3 ./downloads/gallery3
```

### 步骤2：运行批量处理
```bash
python getimgfromurl.py --list my_urls.txt --timeout 30000 --ext jpg
```

### 步骤3：查看结果
程序会：
1. 依次处理每个URL
2. 将图片保存到对应的输出目录
3. 显示详细的处理统计
4. 生成汇总报告

## 错误处理

批量处理具有鲁棒的错误处理：
1. **单个任务失败**：不会影响其他任务的执行
2. **网络问题**：超时或连接失败会记录并继续
3. **无效URL**：会记录错误并跳过
4. **文件权限问题**：会记录错误并继续

## 与单URL模式的兼容性

批量处理模式与单URL模式完全兼容：
- 所有命令行参数（--timeout, --ext, --proxy, --user-agent, --verbose）在两种模式下都适用
- 当指定--list参数时，URL和output_dir参数会被忽略
- 程序会自动检测模式并选择相应的处理逻辑

## 性能优化建议

1. **合理设置超时**：根据网站响应速度调整--timeout参数
2. **任务间延迟**：程序自动在任务间添加2秒延迟，避免被屏蔽
3. **并发限制**：当前版本为顺序处理，未来可考虑并发处理
4. **资源管理**：每个任务完成后会自动清理浏览器资源
