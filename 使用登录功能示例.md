# getimgfromurl.py 登录功能使用示例

## 概述

`getimgfromurl.py` 现已支持网站登录功能。通过提供认证配置文件 (`auth.json`)，脚本可以自动登录网站，然后执行图片抓取任务。

## 认证配置文件格式

创建 `auth.json` 文件，格式如下：

```json
{
    base_url: "https://www.example.com",
    user_name: {selector: "#username", value: "your_username"},
    password: {selector: "#password", value: "your_password"},
    login_button: {selector: "#login_button"}
}
```

### 字段说明

- `base_url`: 网站的基础URL（登录页面）
- `user_name`: 用户名输入框的配置
  - `selector`: CSS选择器，用于定位用户名输入框
  - `value`: 要输入的用户名
- `password`: 密码输入框的配置
  - `selector`: CSS选择器，用于定位密码输入框
  - `value`: 要输入的密码
- `login_button`: 登录按钮的配置
  - `selector`: CSS选择器，用于定位登录按钮
- `wait`: （可选）提交登录请求后需要等待的毫秒数。无论登录是否成功，都会等待指定的时间后再执行后续操作。默认值为0（不等待）

### 选择器说明

- 使用CSS选择器定位页面元素
- 可以通过浏览器开发者工具获取选择器
- 示例：
  - `#username` - 选择id为"username"的元素
  - `.login-form input[type="text"]` - 选择登录表单中类型为text的输入框
  - `input[name="password"]` - 选择name属性为"password"的输入框

## 使用方法

### 1. 单URL模式（带登录）

```bash
python getimgfromurl.py https://example.com/private-page ./output --auth-file auth.json
```

### 2. 批量模式（带登录）

```bash
python getimgfromurl.py --list urls.txt --auth-file auth.json
```

### 3. 带其他参数的使用示例

```bash
# 使用自定义超时时间
python getimgfromurl.py https://example.com/private-page ./output --auth-file auth.json --timeout 60000

# 使用代理
python getimgfromurl.py https://example.com/private-page ./output --auth-file auth.json --proxy http://proxy.example.com:8080

# 启用详细日志
python getimgfromurl.py https://example.com/private-page ./output --auth-file auth.json --verbose

# 批量处理，并行任务数为5
python getimgfromurl.py --list urls.txt --auth-file auth.json --parallel 5
```

## 工作流程

### 单URL模式：
1. **导航到目标URL**：脚本首先导航到指定的URL
2. **检查认证配置**：如果提供了 `--auth-file` 参数，加载认证配置
3. **执行登录**：
   - 导航到 `base_url`
   - 等待登录表单元素加载
   - 填写用户名和密码
   - 点击登录按钮
   - 等待登录完成
4. **验证登录状态**：检查是否成功登录
5. **执行图片抓取**：登录后执行原有的图片抓取流程

### 批量模式（带 `--auth-file` 参数）：
1. **初始化共享浏览器会话**：创建浏览器实例和页面
2. **执行登录**（仅一次）：
   - 导航到 `base_url`
   - 等待登录表单元素加载
   - 填写用户名和密码
   - 点击登录按钮
   - 等待登录完成
3. **验证登录状态**：检查是否成功登录
4. **并行处理所有任务**：使用同一个登录会话处理列表中的所有URL
5. **保持会话状态**：所有任务共享同一个浏览器会话，避免重复登录

### 批量模式（无 `--auth-file` 参数）：
1. **并行处理所有任务**：每个任务创建独立的浏览器实例
2. **无登录步骤**：直接执行图片抓取流程

## 错误处理

- 如果登录失败，脚本会记录错误并继续执行（尝试不登录状态下的抓取）
- 如果选择器找不到对应元素，会超时并记录错误
- 登录状态验证失败不会中止流程，但会记录警告

## 调试技巧

### 1. 验证选择器

在浏览器开发者工具中测试选择器：
```javascript
// 在浏览器控制台中测试
document.querySelector("#username")  // 应该返回用户名输入框
document.querySelector("#password")  // 应该返回密码输入框
document.querySelector("#login_button")  // 应该返回登录按钮
```

### 2. 启用详细日志

使用 `--verbose` 参数查看详细日志：
```bash
python getimgfromurl.py https://example.com/private-page ./output --auth-file auth.json --verbose
```

### 3. 检查登录状态

脚本会自动检查登录状态，但您也可以手动验证：
- 登录后检查页面是否包含"logout"、"sign out"等关键词
- 检查URL是否从登录页面跳转

### 4. 使用截图调试

如果登录失败，脚本会自动保存截图到 `login_test_screenshot.png`，可以查看页面状态。

## 示例配置

### 示例1：简单登录表单

```json
{
    base_url: "https://www.example.com/login",
    user_name: {selector: "#email", value: "user@example.com"},
    password: {selector: "#pass", value: "mypassword123"},
    login_button: {selector: "#submit"}
}
```

### 示例2：复杂选择器

```json
{
    base_url: "https://www.example.com/auth",
    user_name: {selector: "form.login-form input[type='text']", value: "myusername"},
    password: {selector: "form.login-form input[type='password']", value: "mypassword"},
    login_button: {selector: "form.login-form button[type='submit']"}
}
```

## 注意事项

1. **安全性**：`auth.json` 包含敏感信息，请勿提交到版本控制系统
2. **选择器稳定性**：网站更新可能导致选择器失效，需要定期维护
3. **网络条件**：较差的网络连接可能导致登录超时，可适当增加 `--timeout` 值
4. **反爬虫措施**：某些网站可能有反爬虫机制，登录可能失败
5. **会话保持**：登录状态在同一个浏览器会话中保持，但不同任务之间不共享会话

## 向后兼容性

- 不提供 `--auth-file` 参数时，脚本行为与之前完全一致
- 所有原有参数和功能保持不变
- 新增的认证功能是可选的，不影响现有工作流程

## 故障排除

### 常见问题

1. **登录失败，选择器找不到元素**
   - 检查选择器是否正确
   - 检查页面是否完全加载
   - 尝试增加超时时间

2. **登录成功但立即跳回登录页面**
   - 检查登录后是否有验证码等额外步骤
   - 检查会话是否被服务器拒绝

3. **登录成功但无法访问目标页面**
   - 检查目标页面是否需要额外权限
   - 检查登录后是否有重定向问题

### 获取帮助

如果遇到问题，可以：
1. 启用 `--verbose` 模式查看详细日志
2. 检查生成的错误信息和截图
3. 确保网络连接正常
4. 验证网站是否可以正常访问
